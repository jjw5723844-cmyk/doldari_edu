컴퓨터공학 비전공자 교육학 전공자인 저는 취미 겸 포트폴리오로 교육관련 사이트나 프로젝트를 제작하고 있습니다.
저는 제가 앞으로 제작할 프로그램들의 주 언어로 루비를 선택했고 서브 언어로 파이썬을 선택했습니다. 그 이외에도 관심을 가지는 언어에 따라 꾸준히 학습할 예정입니다.
특히 다양한 언어들이 있지만 루비의 경우 개발자와 인간 친화적인 언어라는 철학답게 비전공자나 교육에 종사하는 분들이 파이썬과 마찬가지로 배우고 익히기 쉬운 언어라고 생각합니다.

여기서 잠시 교육학 전공자의 시선으로 루비와 파이썬을 바라보겠습니다.
1. 둘 모두 객체지향 언어이고

2. 해당 언어들은 해당 언어를 이용하는 사람들의 애정어린 손길이 지속적으로 작동하고 있습니다.
루비는 소수정예이지만 해당 언어를 공부하는 이들의 애착이 매우 크기 때문에 오프라인, 온라인 교류가 굉장히 활발합니다.
파이썬은 그 위용만큼이나 기여자들이 많고 학술분야 등 많은 자료들이 파이썬을 통해서 생태계가 이루어 지는만큼 올라운더라고 볼 수 있는 것이죠.
   
3. 두 언어는 모두 교육철학적인 면에서 상당히 공감할 수 있는 방향성을 가지고 있습니다.
루비는 루비는 프로그래머의 행복을 중요시합니다. 이것을 교육학에 대입한다면 교육에 종사하는 프로그램의 제작자나 기획자 등 콘텐츠 제작자들이 배워볼 경우 빠르게 적응할    수 있습니다. 특히 루비의 코드 문단을 끝맺음 하는 end 구문은 정말이지 가독성면에서 쾌적하고 환상적입니다. 개인적으로 파이썬과 비교해봤을때는 훨씬 가독성이 괜챃다고 생각합니다. 또한 파이썬보다 LMS 등 교육관련 운영프로그램들이 국내나 해외에서 다수 이용이 되는만큼 교육에 종사하시는 분들에게는 큰 이점을 가진다고 생각합니다.
물론 파이썬은 2번에서 설명했다시피 올라운더입니다. 교육프로그램에서도 학술적인 부분에서도 파이썬은 사용되지않는 분야가 없습니다. 대학원 과정을 이수하시는 분들이라면 파이썬을 한번쯤은 겪어봤을거라고 생각합니다. 통계와 데이터 등 교육적으로 무궁무진하게 활용가능한 것이 파이썬이지요.

그리고 이 저장소에서 올리는 프로젝트들은 모두 교육적 의미를 담고 제작하였습니다.
물론 하드코딩만 하는 것이 아닌 바이브 코딩도 곁들여서 만들고 있으며, AI의 불안전성을 우려해서 디버그나 검토 등은 수작업으로 하고 있습니다.
계속해서 발전하는 교육의 분야는 이러한 문명의 이기를 적극적으로 사용하지 않으면 학습자나 교육자가 새로운 지식을 받아들이는데 있어서 어려움을 겪을거라고 생각합니다.

이 저장소는 개인용 프로젝트이기도 하지만 교육에 종사하시는 분들 중 프로그래밍에 관심있으신 분들이 적극적으로 구석구석 살펴봐주시길 기원하고 있습니다.

이 프로젝트는 루비(Ruby)의 경량 웹 프레임워크인 **시나트라(Sinatra)**를 활용하여 제작된 교육 및 참고용 웹사이트 예제입니다. 
웹 개발의 기초적인 라우팅(Routing) 구조와 뷰 템플릿(ERB) 연동 방식을 직관적으로 이해할 수 있도록 설계되었습니다.

교육에 종사하시는 분들이나 웹 프로그래밍을 처음 학습하는 학생들이 가볍게 실행해보고 코드를 분석하는 레퍼런스로 자유롭게 활용하실 수 있습니다.

## ✨ 프로젝트 특징 (교육용 포인트)

1. **Zero-Configuration (무설정 작동):** - 복잡한 데이터베이스(MySQL, PostgreSQL 등)를 설치하고 설정할 필요가 없습니다.
   - 서버 내부의 전역 변수(In-memory)를 활용해 데이터가 작동하므로, 다운로드 후 명령어 한 줄로 즉시 실행됩니다.
2. **명확한 MVC 스타일 라우팅:**
   - `app.rb` 파일 하나에 주요 페이지의 주소(URL) 정의와 비즈니스 로직이 깔끔하게 정리되어 있어 웹 서버의 구동 원리를 배우기 좋습니다.
3. **가벼운 프레임워크 환경:**
   - 무거운 레일즈(Rails) 대신 시나트라를 사용하여 코드가 단순하고 직관적입니다.

## 🛠️ 사전 준비 (Prerequisites)

이 프로젝트를 실행하려면 컴퓨터에 아래 프로그램들이 설치되어 있어야 합니다.
- **Gem** (루비의 다양한 기능을 구동할 수 있도록 도와주는 패키지)
- **Ruby** (3.0 버전 이상 권장)
- **Bundler** (루비 패키지 관리 도구)
  ```bash
  gem install bundler


🚀 실행 방법 (Getting Started)
1. 프로젝트 다운로드 및 이동
GitHub에서 코드를 다운로드(또는 Clone)한 후, VSCODE 내부의 터미널(터미널/명령 프롬프트)을 열어 해당 폴더로 이동합니다.

2. 의존성 라이브러리 설치
Gemfile에 등록된 필요한 라이브러리(Sinatra, ERB 등)를 한 번에 설치합니다.

Bash
bundle install

3. 웹 서버 실행
config.ru 설정을 기반으로 서버를 구동합니다.

Bash
bundle exec rackup
(또는 개발 환경에서 코드 수정 시 자동으로 서버가 재시작되도록 하려면 아래 명령어로 실행해도 좋습니다.)

Bash
ruby app.rb

4. 웹사이트 접속
서버가 성공적으로 켜지면 웹 브라우저를 열고 터미널에서 표기되는 두 개의 주소들 중 하나를 선택해서 누르면 사이트에 접속됩니다.


📂 주요 파일 구조 설명
app.rb: 웹 서버의 메인 컨트롤러 파일입니다. 각 페이지의 URL 경로(Route)와 데이터 전달을 담당합니다.

config.ru: 랙(Rack) 기반의 서버 실행 설정 파일입니다.

Gemfile / Gemfile.lock: 프로젝트에 사용된 라이브러리 목록과 버전 정보입니다.

database/: (추가 필요) 가상의 강의 데이터 및 커뮤니티 데이터가 저장된 루비 파일들이 들어갑니다.

views/: (추가 필요) 웹 화면을 구성하는 HTML/ERB 템플릿 파일들이 들어갑니다.

public/: (추가 필요) 이미지, CSS, 자바스크립트 등 정적 파일들이 들어갑니다.


⚠️ 안내 사항 (구현 범위)
현재 이 프로젝트는 교육 및 구조 참고용 예제입니다.

웹사이트의 일부 상세 기능(예: 실제 데이터베이스 저장, 보안 토큰 검증, 일부 미구현 메뉴 등)은 실제 서비스 수준으로 구현되어 있지 않으며, 이는 코드를 내려받아 직접 커스텀해보는 실습 과제로 활용하기 좋습니다.

테스트 모드로 동작하지만 서버를 시작시 제가 직접 입력했던 임의의 데이터들이 존재합니다. 이걸 참고하시면서 직접 한번 자신만의 데이터를 작성해보세요.
